**Location:** Every `<footer>` element across all 13 pages
**Current:** `&copy; 2026 Taylor Stephenscopy; <span id="copy-year">2026</span> Taylor Stephens. All rights reserved.`
**Renders as:** "© 2026 Taylor Stephenscopy; 2026 Taylor Stephens. All rights reserved."
**Fix:** `&copy; <span id="copy-year">2026</span> Taylor Stephens. All rights reserved.`

## Homepage (`index.html`) Issues

### Critical

#### HC-1 · Role/title (tagline) hierarchy inversion
**Location:** `css/styles.css:251–256`
`.tagline` is `0.875rem` in `var(--color-muted)`. `.summary` below it is `clamp(1.125rem, ..., 1.25rem)` in `var(--color-text)`. The job title is smaller and lighter than the supporting prose — a recruiter scanning fast may skip "Strategic Program Manager" entirely.
**Fix:** Elevate `.tagline` to `font-size: 1rem`, `font-weight: 600`, `color: var(--color-heading)`.

### Major

#### HM-1 · No active-page indicator on nav
**Location:** `index.html:30–35`
No `aria-current="page"` and no `.active` class. The sticky nav gives no orientation feedback.
**Fix:** Add `aria-current="page"` to the appropriate nav link and a CSS rule showing the underline by default for that state.

#### HM-2 · Skills grid 4-column layout overflows content at 720px container
**Location:** `css/styles.css:577`
Math: effective column width = `(720 - 48 - 120) / 4 = 138px`. Longest item "Cross-Functional Program Leadership" = 35 chars × ~8.5px = **297px** — more than 2× the column width. Items wrap to 3+ lines, creating jagged uneven rows.
**Fix:** Change to `repeat(2, 1fr)` at desktop, or `repeat(auto-fit, minmax(180px, 1fr))`.

#### HM-3 · CTA hover uses hardcoded `#000`
**Location:** `css/styles.css:626`
`.cta-button:hover { background-color: #000; }` — violates token system.
**Fix:** `background-color: var(--color-heading);`

#### HM-4 · Nav has 5 items; "Personal" uses creator vocabulary
**Location:** `index.html:30–35`
5 items exceeds the 4-item ideal. "Personal" is meaningless to visitors; "Skills" is redundant as a top-level destination.
**Fix:** Remove "Skills" from nav. Rename "Personal" → "Side Projects".

#### HM-5 · Touch target regression at tablet breakpoint
**Location:** `css/styles.css:539–542`
`.contact-links a { min-height: auto; }` at 640px+ removes the 44px touch target set for mobile. Tablets are touch devices.
**Fix:** Remove the `min-height: auto` override.

### Minor

#### Hm-1 · `line-height: var(--baseline)` is a fixed rem value, not a ratio
**Location:** `css/styles.css:90`
`body { line-height: var(--baseline); }` where `--baseline: 1.5rem` = 24px fixed. Footer at `0.75rem` (12px) inherits 24px line-height → 2.0 ratio (too loose).
**Fix:** Change `--baseline: 1.5rem` to `--baseline: 1.5` (unitless multiplier). Update `<p>` `margin-bottom` to remain a fixed rem value.

#### Hm-2 · Header not sticky at tablet (640–1023px)
**Location:** `css/styles.css:551`
`position: sticky` only activates at 1024px+. Long page loses navigation on tablet.
**Fix:** Move sticky header to the 640px breakpoint.

#### Hm-3 · Alternating section backgrounds are container-width, not full-width
**Location:** `css/styles.css:299–304`
`#about` and `#skills` have both `max-width: 720px` and `background-color: var(--color-bg-alt)`. The tinted background appears as a floating strip at wide viewports.
**Fix:** Apply background to an outer wrapper at full-width; use inner `<div>` for max-width constraint.

### Cosmetic

#### Hco-1 · All shadows are single-layer
`--shadow-sm/md/hover` use one `box-shadow` each. Natural shadows use two layers (ambient + direct).
**Fix:** `--shadow-hover: 0 4px 12px rgba(0,0,0,.10), 0 1px 4px rgba(0,0,0,.06);`

#### Hco-2 · Project cards have no resting shadow
Cards rely solely on borders at rest, only elevate on hover.
**Fix:** Add `box-shadow: var(--shadow-sm)` to `.project-card` base state.

#### Hco-3 · CTA microcopy is passive
`index.html:47` — "View My Work". Consider "See the Case Studies →" (specific).

#### Hco-4 · Contact copy is generic
`index.html:194` — "Open to program management opportunities. Let's connect." is the default portfolio phrase.

#### Hco-5 · Nav focus state missing underline animation for keyboard users
`css/styles.css:601–618` — `nav li a:hover::after` underline doesn't fire on `:focus-visible`.
**Fix:** Add `nav li a:focus-visible::after { transform: scaleX(1); }`
