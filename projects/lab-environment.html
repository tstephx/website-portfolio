Research First, Build Second: Designing a Learning Environment That Compounds

A development environment where every project starts with curated book knowledge before external sources — a system designed to make learning compound.

Claude Code Book Library MCP Markdown Shell Workflow

The Problem

Starting projects by Googling leads to shallow, scattered knowledge. You end up following a random Medium tutorial instead of the Docker book you already own. You know you’ve read about a topic but can’t find where. Each project starts from zero instead of building on what you’ve already learned.

The philosophical issue: learning should compound. Every book read should make the next project easier. But without a system to enforce that, the default behavior is to reach for whatever Google serves first — which is usually not the best source you have available.

The Approach

I designed a workflow environment with one hard rule: book library first, always, no exceptions. Before writing code, searching the web, or asking generic questions, the process requires searching the curated book library, reading relevant chapters, and extracting patterns.

The Golden Rule
BOOK LIBRARY FIRST. Always. No exceptions.

Curated knowledge before search engines — depth over breadth

Three-Step Plan

  • Define research-first workflow: Mandatory research phase before implementation. Every project document has a Research section that must be filled before the Implementation section is touched.
  • Integrate book library as primary source: The Book Library MCP server provides get_topic_coverage(), semantic_search(), teach_concept(), and generate_learning_path() as the first tools Claude reaches for.
  • Template projects to enforce the pattern: Project templates include pre-built research prompts and checklists that guide the research-first approach.

Session Start Protocol

Every session begins with two queries against the book library: get_topic_coverage("[MAIN TOPIC]") to find all relevant chapters, and semantic_search("[KEY CONCEPTS]") to surface conceptual connections. This ensures Claude is primed with relevant knowledge before any work begins.

# Step 1: What do I already know about this?

# Step 2: What concepts are related?

Research-First Core Principle

The environment changed how projects start. Instead of “let me Google that,” the default is “what do my books say?” Cross-referencing multiple authors on the same topic became natural. Learning from previous projects became automatic because the book library serves as institutional memory across all work.

Lessons Learned

  • The constraint of “books first” forces deeper engagement with material — you read more carefully when you know it’s your primary source, not a backup.
  • Templates reduce friction between intention and action — a pre-built research checklist makes the right behavior the default behavior.
  • Compounding knowledge requires a system, not willpower. The environment enforces the habit; the habit builds the knowledge.
